⚙️ 7. Process Management — Управление процессами

7.1 Background / Foreground Processes — Фоновые / активные процессы

Подтемы:
•	Запуск в фоне:
•	command &
•	Вывести процесс на передний план:
•	fg
•	Перевести в фон:
•	Ctrl + Z → bg
•	Просмотр фоновых задач:
•	jobs
•	Отличие jobs от ps

⸻

7.2 Listing / Finding Processes — Просмотр / поиск процессов

Подтемы:
•	ps — подробная информация:
•	ps aux — все процессы
•	ps -ef — альтернативный формат
•	top — интерактивный мониторинг
•	htop — улучшенный top
•	Поиск процесса:
•	pgrep name
•	ps aux | grep name

⸻

7.3 Process Signals — Сигналы процессов

Подтемы:
•	Что такое сигналы (SIGTERM, SIGKILL, SIGSTOP)
•	Посмотреть список сигналов:
•	kill -l
•	Основные:
•	SIGTERM (15) — корректное завершение
•	SIGKILL (9) — принудительное уничтожение
•	SIGSTOP — остановка
•	SIGCONT — продолжить
•	Использование:
•	kill -15 PID
•	kill -9 PID

⸻

7.4 Killing Processes — Завершение процессов

Подтемы:
•	kill PID
•	pkill name — по имени
•	killall name — все процессы по имени
•	Завершение зависших процессов
•	Мониторинг:
•	top → k (ввести PID)

⸻

7.5 Process Priorities — Приоритеты процессов

Подтемы:
•	Приоритеты (nice value) — от -20 до 19
•	Проверка:
•	top (поле NI)
•	ps -l
•	Изменение при запуске:
•	nice -n 10 command
•	Изменение уже работающего процесса:
•	renice -n 5 PID
•	Когда повышать/понижать приоритет

⸻

7.6 Process Forking — Форкинг процессов

Подтемы:
•	Что такое fork() — создание нового процесса
•	PID / PPID — идентификатор процесса и родителя
•	Деревья процессов:
•	pstree
•	Почему могут появляться “зомби”-процессы
•	Опасность бесконечного форка (fork-bomb)
•	Пример fork bomb (только для теории!):
•	:(){ :|:& };:
